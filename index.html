js中关于数的犄角旮旯

发表于 2018-05-06

数字Number

IEEE二进制浮点数算术标准

浮点数的实际值 = 符号位 * 指数偏移值 * 分数值 指数偏移值 = 指数编码值 - 固定值 固定值 = 2^(e - 1) - 1 e = 指数域的比特位数

JS中的数字表示

JS中Number采用64位二进制数表示，其中1位符号位，11位指数
位以及52位有效数据位。

JS中还有一些特殊值的表示如下表所示

52位有效数字加上整数部分默认隐含的1（见由IEEE 754对“规约形式的浮点数”的规定）能表示的最大的数是2^53 - 1，JS中能精确表示到个位的最大整数是2^53 - 1，内存模型见下图1。再往后加1是2^53，内存模型如下图2所示。

从2^53开始，第一个“尾数”被舍弃，每两个值都有一个不精确，准确的说是2^53<= n <2^54中的数会出现每2个值有一个不精确，2^54<= n <2^55中的数会出现每4个值有3个不精确，2^55<= n <2^56中的数会出现每8个值有7个不精确，总结出来就是2^(52+k)<=n<2^(52+k+1)中的数会出现每2^k中会有2^k-1个数不精确。 对照下图（来自大神的文章）比较好理解。64位双精度浮点数表示法中，有效数字只有52位，能表示的最大的精确的数是2^53 - 1（就是Number.MAX_SAFE_INTEGER），2^53<= n <2^54之间的数字的表示靠的是指数的进位，此时指数进一位，代表有效数字部分进两位，因此可以想像成如下图所示的最后一个“有效数字位”为0，并且只能为0，最后的“有效数字”只能被舍弃，无法表示，因此每两个数字会出现一个数字不精确 。依此类推，当指数部分进两位时，代表有效数字部分进四位，如下图所示的最后两个“有效数字”为0，无法表示，因此每四个数字会有三个数字不精确。

JS中能表示的最大整数是(2^53 - 1)*2^(2046-1023-52)，也就是Number.MAX_VALUE表示的值。能表示的最小正数是2^(1-1023)*2^(-52)，也就是Number.MIN_VALUE表示的值。内存模型分别如下图所示。

JS中的+0和-0

按照第一部分说的JS中Number的数字的表示法，可以推想到JS中会存在分别带“+”和“-”的两个0。平常使用时不加符号的话默认是“+0”。

  • +0和-0用“===”做比较时得到的结果是true，无法用“===”做区分
  • “<”和“>”也无法区分两者

如何区分两个0

  • 用任意正数除以+0得到的结果是+Infinity，反之得到-Infinity。+Infinity和-Infinity可以直接用“===”比较。

  • 用JS中现有的方法区分。JS中有用到一种Same-value的比较算法，这种比较算法可以用来做“真正严格”意义上的比较。Set结构中不能添加重复的值，其中所使用的就是Same-value比较算法。还有object.is()，判断两个值是否相等（当然也可以判断NaN）。还有一种比较骚气的解法，使用Object.defineProperty()，在试图修改不可改变的属性时，如果值确实修改了，将会抛出异常。代码如下。（此文档中有JS中区别对待-0和+0操作的不完全列表）
